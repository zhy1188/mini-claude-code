# 每日改动报告规范

每次完成一组相关改动后，自动写进 `docs/今日改动-YYYY-MM-DD.md`。

## 筛选标准

- 每部分选出 **3-5 个核心点**
- 按面试时可讲性排序：**问题 → 设计思路 → 关键代码 → 面试话术**
- 每个点都要能讲清楚"为什么这么做"和"技术决策的权衡"

## 报告模板

```markdown
# 今日改动 - YYYY-MM-DD

## 一、[改动主题]

### 1. [核心点名称]

**核心代码路径**：`src/xxx/yyy.py`

**问题**：原有方案有什么缺陷，为什么需要改。

**设计思路**：怎么解决的，关键技术决策是什么。

```python
# 关键代码片段（10行以内，展示核心逻辑）
```

**面试讲述**：用 1-2 段话把问题、方案、权衡讲清楚，附带量化效果。
```
